Analysing Twitter Posts to Discover and Review New Software Tools

Twitter is one of the most active microblogging social networks in use today. Millions of users post 140 character snippets every day to Twitter, many of which express opinions about different topics. Many organisations and institutions find this information interesting, be it in biomedicine, marketing or the financial sector, and as such has led to several research projects focussed on text mining Twitter.

Text mining is the process of extracting useful information from unstructured texts by means of structuring the data, deriving patterns from the structured text and then interpreting the output.

The aim of this project is to text mine Twitter posts to find new software or tools, such as software development tools, and to discover public perception of these releases.

Twitter is being used as a corpus as it is a rich source of data for opinion mining and sentiment analysis with over 200 million tweets every day.

Technologies Used

Java + Eclipse

Java is my choice of programming language for this task as there are already many libraries available for the task at hand. It is also the language I feel most comfortable working with due to my previous experiences with it.

I have been using the Eclipse IDE due to its ease of use and also because I have used it before for many of my own projects.

Python + NLTK(Natural Language ToolKit)

I have decided to use Python in this project to work with NLTK. NLTK is the Natural Language ToolKit developed for all sorts of natural language processing tasks, including but not limited to tokenization, classification and sentiment analysis. I chose to use this toolkit ahead of others such as MinorThird and GATE due to its extensive documentation and for the sheer power the Python programming language possesses. Python is a language I have had to learn solely for this task in my project, but I made this decision to provide a better grounding in other technologies, rather than limit myself to Java.

MySQL

I am using MySQL to store my all of my data as it is readily available on the school's computers and I have worked with it on many occasions in the past.

The database for this project stores users of the Twitter service, with all of their tweets, as well as tools discovered on the site as can be seen below.

Twitter's Streaming API

Twitter provide 3 APIs for developers. The standard REST API, Search API and the Streaming API.

The Streaming API provides near-realtime access to public tweets filtered in various ways. I have chosen to filter these tweets by keywords that are somewhat likely to appear in software-related tweets, such as names of operating systems and programming languages.

Twitter Sentiment

Twitter Sentiment provides a bulk classification service for analysing the sentiment of texts. The service is free of charge, with no limit, except that there is a maximum number of 10,000 texts you can have analysed at a time.

I am using this service ahead of others such as AlchemyAPI, which offers clients many services including but not limited to sentiment analysis, named entity recogition and concept tagging, because it allows unlimited use, whereas AlchemyAPI only allows clients to make 30,000 API calls per day across the service.

Key Tasks

Objectives:

  1. Collect tweets.
  2. Filter through tweets initially by keyword to find software-related posts. E.g. names of software, operating systems, games.
  3. Create linguistic preprocessing tools.
  4. Analyse tweet sentiment.
  5. Structure and integrate data.
  6. Visualise data through GUI.
  7. Evaluate system.

Deliverables:

  1. Integrate Twitter Streaming API into stub application.
  2. MySQL database with tables holding Twitter users, tweets and discovered tools.
  3. Named-entity recogniser for tools.
  4. Create or integrate existing sentiment classifier.
  5. Improve information retrieval and filtering algorithms based on data.
  6. Produce GUI
